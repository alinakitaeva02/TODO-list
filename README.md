# TODO лист

## Описание проекта

TODO лист - веб-сервер для ведения списка задач. Приложение позволяет создавать, редактировать, удалять и отмечать задачи выполненными, хранит данные в SQLite и предоставляет браузерный интерфейс.

Для повторяющихся задач сервис рассчитывает следующую дату выполнения по правилу повторения. Одноразовые задачи после выполнения удаляются.

## Выполненные задания со звездочкой

Выполнялись задания повышенной трудности:

1. Определение пути к файлу базы данных через переменную окружения `TODO_DBFILE`;
2. Расширенные правила повторения задач;
3. Возможность выбрать задачу через строку поиска;
4. Создание Docker-образа.

## Локальный запуск

Требования:

- Go 1.25.5 или новее.
- Свободный порт `7540`.

Запуск с базой данных по умолчанию:

```bash
go run ./main.go
```

После запуска откройте в браузере:

```text
http://localhost:7540
```

По умолчанию база данных создается в файле `scheduler.db` в корне проекта. Чтобы указать другой путь, задайте переменную окружения `TODO_DBFILE`.

Пример для PowerShell:

```powershell
$env:TODO_DBFILE="C:\Users\Алина\Dev\final_project\scheduler.db"
go run ./main.go
```

Пример `.env`:

```env
TODO_DBFILE=./scheduler.db
```

## Docker

Dockerfile собирает Linux-бинарник в отдельном build-слое, а в финальный образ копирует только исполняемый файл и поддиректорию `web`. Порт сервера указан через `EXPOSE 7540`. Переменная `TODO_DBFILE` задана в образе как `/app/data/scheduler.db`.

Сборка образа:

```bash
docker build -t todo-scheduler .
```

Запуск контейнера с подключением SQLite-базы на хосте:

```bash
mkdir data
docker run --rm -p 7540:7540 -v ${PWD}/data:/app/data todo-scheduler
```

В этом варианте контейнер использует файл базы `/app/data/scheduler.db`, который физически хранится в локальной папке `data` на хосте.

Если нужно указать другой путь внутри контейнера, передайте `TODO_DBFILE` явно:

```bash
docker run --rm -p 7540:7540 -e TODO_DBFILE=/app/data/scheduler.db -v ${PWD}/data:/app/data todo-scheduler
```

После запуска откройте:

```text
http://localhost:7540
```
